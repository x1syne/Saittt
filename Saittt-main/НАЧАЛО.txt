🚀 ЧТО НУЖНО СДЕЛАТЬ СЕЙЧАС (5 МИНУТ):

┌──────────────────────────────────────────────────────────────────┐
│ ШАГ 1: НАСТРОЙТЕ SPOTIFY APP (2 минуты)                         │
└──────────────────────────────────────────────────────────────────┘

  1. Откройте: https://developer.spotify.com/dashboard
  2. Выберите ваше приложение (или создайте новое)
  3. Нажмите "Edit Settings"
  4. В "Redirect URIs" добавьте:
     
     https://saittt.vercel.app/auth/spotify/callback
     
  5. Нажмите "Add" и "Save"

┌──────────────────────────────────────────────────────────────────┐
│ ШАГ 2: НАСТРОЙТЕ VERCEL (2 минуты)                              │
└──────────────────────────────────────────────────────────────────┘

  1. Откройте: https://vercel.com/dashboard
  2. Выберите проект "saittt"
  3. Settings → Environment Variables
  4. Добавьте 3 переменные:
  
     SPOTIFY_CLIENT_ID = [из Spotify Dashboard]
     SPOTIFY_CLIENT_SECRET = [из Spotify Dashboard]
     REDIRECT_URI = https://saittt.vercel.app/auth/spotify/callback
     
  5. Deployments → Redeploy последнего деплоя

┌──────────────────────────────────────────────────────────────────┐
│ ШАГ 3: ЗАГРУЗИТЕ НА GITHUB (1 минута)                           │
└──────────────────────────────────────────────────────────────────┘

  ⚡ БЫСТРЫЙ СПОСОБ:
  
  1. Найдите файл: setup-and-push.bat (в корневой папке)
  2. Дважды кликните по нему
  3. Следуйте инструкциям
  
  📝 ИЛИ ВРУЧНУЮ:
  
  git remote add origin https://github.com/x1syne/Saittt.git
  git push -u origin main

🆘 ЕСЛИ НЕ РАБОТАЕТ:

  ❌ "INVALID_CLIENT"
     → Проверьте Client ID и Secret в Vercel
     → Сделайте Redeploy
  
  ❌ "Redirect URI mismatch"
     → В Spotify должно быть точно:
       https://saittt.vercel.app/auth/spotify/callback
  
  ❌ "Token not found"
     → Откройте консоль браузера (F12)
     → Проверьте ошибки
  
  ❌ CORS ошибки
     → Убедитесь, что backend задеплоен на Vercel
